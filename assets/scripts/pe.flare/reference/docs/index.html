1.2 Setting Up The Plugin


Copy or upload the js/pe.flare folder somewhere onto your server, for example into the top level js folder. To save space, you can remove the src sub folder which is only needed if you wish to modify the javascript source code.

Include the following javascript and css code into HEAD section of the page
<!-- jQuery (only needed if your page doesn't already load it ) -->
<script type="text/javascript" src="js/jquery.js"></script>


<!-- flare lightbox css -->
<link href="js/pe.flare/jquery.pixelentity.flare.min.css" rel="stylesheet"/>
<!-- flare lightbox javascript -->
<script type="text/javascript" src="js/pe.flare/jquery.pixelentity.flare.min.js"></script>
Next see the following section for how to activate the plugin and attach it to your page elements.


Make sure when uploading the files to your server that the inner folder structure of the above folder is maintained, otherwise the plugin will not function as expected.



1.3 Activate the plugin

Flare can be activated by placing the following code anywhere in your page BODY section:
<script type="text/javascript">
        jQuery(function($){
                $('a[data-target="flare"]').peFlareLightbox();
        });
</script>
The above code searches for all <a> tags which have a data-target attribute set to "flare" and once found, it activates the lightbox plugin for those links.

2 The Responsive Lightbox Usage Examplestop

Flare is a custom responsive, touch enabled, mobile optimized lightbox which can be used to display single images/videos or entire galleries. Items can be browsed using buttons, swipe gestures on mobile and mousewheel/keyboard on desktops. It uses hardware accelerated css transforms and fallbacks to standard jquery animation with in older browsers. It supports youtube, vimeo, vid.ly and local videos playback. In this section you will find examples of how to take full advantage of all of its features.
For simplicity and readability sake, text links are used in the following examples as the "triggering" element. Meaning the lightbox would be launched once the hyperlink is clicked. These can be changed to images/thumbnails easily by including an <img> tag inside the parent <a> tag with the data-target="flare" attribute. For examples of using thumbnails instead of text links, see the examples.html file included in the download package.

2.1 Single image

<a
     data-target="flare"
     href="image.jpg"
     data-flare-scale="fit"
     >


    Single Image</a>
data-targetWhen sets to "flare", activates the lightbox when user clicks on the item
data-flare-scaleControls the image scaling, possible values are

"fit" - Fits the window: no cropping, always show entire image (default mode)
"fill" - Fills the window: horizontal/vertical cropping may occur when image and window aspect ratios are different (almost always)
"fitmax" - Same as "fit" but upscales the image if smaller than window
"fillmax" - Same as "fill" but upscales the image if smaller than window

2.2 Single Online Video

<!-- YouTube -->
<a data-target="flare" href="http://www.youtube.com/watch?v=AFA-rOls8YA">Youtube</a>
<!-- Vimeo -->
<a data-target="flare" href="http://vimeo.com/39251390">Vimeo</a>
<!-- Vid.ly -->
<a data-target="flare" href="http://vid.ly/5u4h3e">Vid.ly</a>
As you can see from the above markup, you only need to link the video location: the cover image will be fetched automatically from the appropriate source.
The Video will be shown fullscreen

2.3 Single Local Video

<!-- Local Video -->
<a 
     data-target="flare" 
     data-flare-videoposter="video/poster.jpg" 
     data-flare-videoformats="webm,ogv" 
     href="video/video.mp4"
     >
    Local Video</a>
Unlike the previous case, local video playbacks requires a few extra settings.

href is the url of a mp4 video hosted on your site (local video example)
data-flare-videoposteris the video cover image
data-flare-videoformats define the additional formats you need to encode your video in order to use html5 video playback in all browsers. This basically means you should have on your server:

video.mp4
video.webm
video.ogv
The Video will be shown fullscreen

2.4 Gallery


Multiple images can be grouped in a gallery: when one of them is being displayed, it will be possible to navigate all of them without closing the lightbox. Navigation can happens in multiple ways: prev/next buttons, swipe gestures, mousewheel and keyboard. To group images in galleries, just add the data-flare-gallery attribute to the markup:
<a data-target="flare" data-flare-gallery="gallery1" href="01.jpg">Image 1</a>
<a data-target="flare" data-flare-gallery="gallery1" href="02.jpg">Image 2</a>
<a data-target="flare" data-flare-gallery="gallery1" href="03.jpg">Image 3</a>
All the above images will be grouped in "gallery1", you can have different galleries in the same page
Image navigation methods supported:

  1. Next / Previous UI Arrows
  2. Keyboard Left / Right Arrows
  3. Mouse Wheel Scroll
  4. Swipe Gestures (On Mobile & Tablet Devices)

2.5 Gallery with thumbnails


Gallery mode can also display thumbnails of all images in the bottom lightbox area, to activate the thumbnails, use the data-flare-thumb attribute
<a data-target="flare" data-flare-gallery="gallery1" data-flare-thumb="thumb01.jpg" href="01.jpg">Image 1</a>
<a data-target="flare" data-flare-gallery="gallery1" data-flare-thumb="thumb02.jpg" href="02.jpg">Image 2</a>
<a data-target="flare" data-flare-gallery="gallery1" data-flare-thumb="thumb03.jpg" href="03.jpg">Image 3</a>
Image navigation methods supported:

  1. Next / Previous UI Arrows
  2. Selelect a Thumbnail
  3. Keyboard Left / Right Arrows
  4. Mouse Wheel Scroll
  5. Swipe Gestures (On Mobile & Tablet Devices)
If thumbs are showing, swipe gestures will control the thumbnail scrolling. Close the thumbnails to use swipe gestures on the main images.

2.7 Alternate Gallery - Shutter Effect


Flare also includes an alternate gallery mode using a black and white/zooming image transition: this mode is called shutter and can be activated using the data-flare-plugin attribute
<a
     data-target="flare"
     data-flare-plugin="shutter"
     data-flare-gallery="newGalleryName"
     data-flare-scale="fillmax"
     data-flare-thumb="img/content/small/01.jpg"
     data-flare-bw="img/content/big/01bw.jpg"
     href="img/content/big/01.jpg"
     >


    Fullscreen Shutter Gallery With Thumbs - Image 1

</a>
<a
     data-target="flare"
     data-flare-plugin="shutter"
     data-flare-gallery="newGalleryName"
     data-flare-scale="fillmax"
     data-flare-thumb="img/content/small/02.jpg"
     data-flare-bw="img/content/big/02bw.jpg"
     href="img/content/big/02.jpg"
     >


    Fullscreen Shutter Gallery With Thumbs - Image 2

</a>
Since shutter mode requires a black and white version of the image to do the transition, we need the extra parameter data-flare-bw to define it. The bw image is not mandatory and can be omitted, obviously in this case, the transition would skip the black and white effect part.
Black & White version of the main image and also the thumbnail image need to be prepared manually. this plugin does not automate this process.

2.8 Gallery From Hidden Items


You can include items in a gallery which are not displayed on the page. To do this simply make sure the initial item in the gallery is shown on the page as normal, then create a div in the page somewhere, add a class of .hiddenLightboxContent to this div element. This class will hide the content placed inside this div but provided you use the same data-flare-gallery="galleryname" as the displayed item, they will be added to that gallery.
<!--displayed item-->
<a data-target="flare" 
   data-flare-gallery="HiddenGallery" 
   href="01.jpg">
Image 1</a>




<!--hidden items-->
<div class="hiddenLightboxContent">
<a data-target="flare"
   data-flare-gallery="HiddenGallery" 
   href="02.jpg">
Image 2</a>


<a data-target="flare"
   data-flare-gallery="HiddenGallery" 
   href="03.jpg">
Image 3</a>


</div>

3 CSS Stylesheets - Organisationtop

Because this component is a plugin and has been designed to use interchangable skins, the organisation of the css stylesheets is a little different than that of a normal website or template. This component uses one stylesheet to control it's layout, margins, padding, heirarchy and basically everything which is not to do with how the item looks on the surface. By this I mean, the colors, buttons, radius of rounded corners, border styles etc. These are all specified in a separate CSS stylesheet, because these are related to the component's skin. The benefits of this organisation is that the skin stylesheet can then be replaced with a new skin stylesheet and the whole look of the component can therefore be easily changed, without having to worry about upsetting the underlying CSS structure of the component.


3.1 Stylesheets


It is important that the folder structure of this component is maintained. All CSS information for the component is contained inside the "themes" folder. Inside the folder is the "common.css" file and then a subfolder for each skin. Each one of these subfolders contains a separate skin stylesheet and an "img" folder. This folder contains all the images which are specific to that skin.

themes/common.cssControls all CSS layout structure, spacing, positioning for the component & it's controls.
themes/default/skin.cssControls the external look of the "default" theme or skin. This includes the buttons, background images, colors, patterns.
This plugin currently only contains one single skin, the default skin.

3.2 Basic Styling 1 - Controls


In the themes/default/skin.css you will find the CSS for the images used in the lightbox controls. To swap these images with your own versions you will need to change the following CSS:
/* Previous Icon */
.peFlareLightbox .peFlareLightboxControls a#peFlareLightboxControlPrev {
    background:url(img/prev.png) no-repeat 0 0;
}


/* Next Icon */
.peFlareLightbox .peFlareLightboxControls a#peFlareLightboxControlNext {
    background:url(img/next.png) no-repeat 0 0;
}


/* Close Icon */
.peFlareLightbox .peFlareLightboxControls a#peFlareLightboxControlClose {
    background:url(img/close.png) no-repeat 0 0;
}


/* Thumbs Icon */
.peFlareLightbox .peFlareLightboxControls a#peFlareLightboxControlThumbs {
    background:url(img/thumbs.png) no-repeat 0 0;
}
In the download package you will find a psd folder which contains the original images for the flare lightbox ui. This file may be of help to you should you decide to insert your own icons.

3.3 Basic Styling 2 - Frames


In the themes/default/skin.css you will find the CSS for the image frames (fit image and thumb image). To edit the styles for these frames you will need to change the following CSS:
/*thumbnail frames*/


.peFlareLightbox .peFlareLightboxThumbs > div > div > span {
    border: 2px solid white;
    box-shadow: 2px 2px 2px #000;
}


/* fit image frame*/


.peFlareLightbox .peFlareLightboxContent .peFlareLightboxRenderImage img.fit {
    border: 4px solid white;
    box-shadow: 4px 4px 4px #000;
}
When editing the CSS of the plugin you will need to account for the fact that by default this plugin used compressed / minified CSS. See the section below for more info. about this.

3.4 Basic Styling 3 - Where To Make CSS Edits


This plugin used compressed CSS because this makes the whole plugin very quick to load. This means however that you cannot modify the CSS styles directly but must overwrite those styles instead. To do this you have 2 options:

  1. Add the edited styles into the head section of your HTML page.
  2. Manually create and insert a new stylesheet. Import this stylesheet into the HTML page immediately after the default CSS is imported. This will ensure overwriting.
NOTE: Sometimes when using the above method for CSS modification, if a CSS style modification does not result in the expected style change even though the CSS selector syntax is correct, you may need to use the !important keyword in your CSS rules. This sill ensure the default rules are overwritten.
Also note that to ensure overwriting of an existing CSS style, the exact same element selector needs to be used. You can find these selectors by inspecting the page, using Chrome element inspector, FireFox Firebug addon, and lots of other developer tools functions in most modern browsers.

4 iOS & iDevicestop

Flare has been optimized to work with Apple's mobile operating system iOS. The site has been tested on iOS5 primarily on the iPad2 and the iPhone 4. To ensure the template works well on iDevices, sometimes the CSS of the template needs to be tweaked or sometimes new rules need to be added to control specific behavior found only on the mobile version of Apple's Safari browser.


4.1 iDevice CSS


This iDevice/mobile device specific CSS can be found at the bottom of the Stylesheets. All iDevice/mobile device styles can be recognized by their .mobile prefix. As example is shown below. This particular rule is required to prevent iOS devices from adjusting the size of the fonts in the template irrespective of the orientation and zoom level.
/* STYLES FOR APPLE DEVICES (iDevices) */
    .mobile body,
    .mobile p,
    .mobile div,
    .mobile h1,
    .mobile h2,
    .mobile h3,
    .mobile h4,
    .mobile h5,
    .mobile h6 {
    -webkit-text-size-adjust: none;
    }

4.2 Rollover Effects on iDevices


Some of the interesting effects used in Flare are seen or experienced by the user only when rolling over an element using their mouse. However when using a touch screen device such as the iDevices, the rollover state does not feature as prominently. It is still possible to see rollover effects as follows. In Apple's mobile Safari browser the way to activate a rollover state for an element is by Soft Clicking. This is simply a normal click, but as long as the element being clicked has a rollover state, then this state will be activated and a second click is then needed in order to visit the link or page or whatever is the default action of the link itself.

Sometimes however, rollover effects have been intentionally switched off in the CSS because they will not function correctly on a touch based device due to the different nature of a touch user interface from that of a mouse based one.



4.3 Flare and Android


Flare has been tested on the Android operating system using a Samsung Galaxy Tab 10.1. running the Honeycomb version of the OS and also a Samsung Galaxy SII phone running the Ice Cream Sandwich version of the OS. While Flare is working fine on the Honeycomb device, the template has been optimized for the latest version of the Android OS which is Ice Cream Sandwich. There are some well known bugs in the Android OS relating to video playback, there feature more so in Honeycomb, but they are a flaw in the default browser of the OS and are not related to the template. Additionally it should be noted that despite having been optimized for Android ICS, this does not mean that this template will work flawlessly on all Android devices. The reasons for this are as follows:

  1. There are just a vast number of different devices from different manufacturers, out there all running android and all with varying hardware specs. Each manufacturer has also tweaked or skinned the OS in their own way. Consequently it is impossible to perform testing on all such devices and to allow for any slight differences there may be in the way each device renders content.
  2. For reasons unknown to us, manufacturers of Android handsets and tablets are extremely slow in deploying updated to the OS. Consequently there are vast numbers of devices out in the wild which are running an array of different versions of the Android OS. Many of these devices may not have the hardware or software to run the template properly and so its impossible for us to promise full compatibility.
It is also worth noting that Android users may not enjoy as good an experience browsing this template as iOS users will. This is because at the time of writing the Android operating system does not allow for the experiencing of rollover states as iOS devices do.
